# DD2417 Project Group 8

Emma Tisell & Alexander Runebou

This repository contains a keystroke-saving word prediction system developed for the DD2417 project.

The system compares two approaches to word prediction:

- an n-gram predictor
- a transformer-based predictor

It also includes a simple spell checker based on edit distance and word frequency.

The goal is to simulate a user typing text and measure how many keystrokes can be saved when the correct word appears among the model's suggestions.

---

## Setup

This project uses `uv` for Python environment and dependency management.

### 1. Install uv

On macOS or Linux:

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
```

On Windows PowerShell:

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```


---

### 2. Create and activate the virtual environment

From the repository root:

```bash
uv venv
```

Activate it.

On Linux or macOS:

```bash
source .venv/bin/activate
```

On Windows PowerShell:

```powershell
.venv\Scripts\Activate.ps1
```

On Windows CMD:

```cmd
.venv\Scripts\activate.bat
```

---

### 3. Install dependencies

If the project contains a `pyproject.toml`, install the dependencies with:

```bash
uv sync
```

## Data setup

Create a `data/` folder if it does not already exist:

```bash
mkdir -p data
```

The training corpus should contain plain text files. For this project, the training data includes:

- the Harry Potter books from previous assignments
- additional books downloaded from Project Gutenberg

## Training

The system uses three trained components:

1. Tokenizer
2. Transformer checkpoint
3. Word counts

The word counts are used by the n-gram predictor, the transformer predictor, and the spell checker.

---

### 1. Train the tokenizer

Example using some files from data:

```bash
python -m training.train_tokenizer \
  --texts data/hp1.txt data/hp2.txt data/hp3.txt data/frankenstein.txt data/hamlet.txt \
  --output model/tokenizer.json \
  --vocab-size 5000
```


---

### 2. Train the transformer

```bash
python -m training.train_transformer \
  --texts data/hp1.txt data/hp2.txt data/hp3.txt data/frankenstein.txt data/hamlet.txt \
  --tokenizer model/tokenizer.json \
  --checkpoint model/checkpoint.pt \
  --epochs 2 \
  --batch-size 8 \
  --block-size 256
```

---

### 3. Train word counts

```bash
python -m training.train_word_counts \
  --texts data/hp1.txt data/hp2.txt data/hp3.txt data/frankenstein.txt data/hamlet.txt \
  --output model/word_counts.pkl
```

---

### 4. N-gram model

The n-gram model does not require a separate checkpoint file. It is built from the training texts during prediction and evaluation.

Make sure the required training texts and `word_counts.pkl` exist before running demos or evaluations.

## Running the demo

Run the graphical demo with:


## Evaluation

Evaluation scripts are located in the `evaluation/` folder.

---

### Word prediction evaluation

To evaluate keystroke savings, make sure `data/corpus_books.txt` exists with all training texts (which can be generated by the training scripts), then run:

```bash
python -m evaluation.evaluate -f data/testing/moby_dick.txt -p ngram
```

---

### Spell checker evaluation

The spell checker is evaluated using `evaluation/eval.csv`. Run:

```bash
python -m evaluation.evaluate_spelling
```
